wsl 2 & ubuntu

I ran into a problem while developing a Django app using Windows Subsystem for Linux. Apparently the permissions for some of the files and folders if you create them in Ubuntu and you are working in vsCode are set to Root and this will not allow you to edit and save your progress. At first I had no idea what EAccess error I was getting was, after some research, found you need to change owner and change group of the folder you are working in with (chown -R [user] [folder/file]) and (chgrp -R [user] [folder/file]) after that you should be able to edit and save your progress. Hope this helps someone that runs into this problem. Happy Coding!!!!

Found Solution

I was able to get my database up and working properly not without some frustrations. When using SQLALCHEMY 1.4 the database URI has to have postgresql:// instead of postgres://, which is what the heroku command line gives you to use when you type the command in to get the path to the database, so that is very frustrating. I had to sign in to heroku in browser destroy the database and go through the process again except when I copy and pasted the path into my app.py file I just changed the beginning of the path to postgresql://, not very complicated just some research and a lot of patience, guess that come with the territory of software development.
